# Sprint 5 — TypeScript Foundations

## Why are design tokens better than hardcoded colors?
because it's easy to maintain suppose if we have to change in future we don't have to go in each and every file to hunt the colors.

We don't name colors.
We name responsibilities.

for example we don't name the color as black
we will name it as primaryColor because if the color changes .. stil the meaning remains same otherwise it's the same thing with hardcoded colors because eventually we have to change color name also ... so hectic...

## Principle 1
Components never own colors. They consume them.
